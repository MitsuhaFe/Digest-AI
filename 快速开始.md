# 快速开始 - Digest AI

## 🚀 5 分钟快速启动指南

### 前置要求

- ✅ Node.js (v16+)
- ✅ npm 或 yarn
- ✅ Chrome 浏览器
- ✅ 一个 AI 模型的 API Key（推荐 Gemini，有免费额度）

### 步骤 1: 安装依赖 (1 分钟)

在项目根目录打开终端：

```bash
npm install
```

### 步骤 2: 创建临时图标 (1 分钟)

**最快方法**：

1. 找任意一张 PNG 图片
2. 在 Windows PowerShell 中运行：

```powershell
# 确保在项目根目录
copy 你的图片.png icons\icon16.png
copy 你的图片.png icons\icon48.png
copy 你的图片.png icons\icon128.png
```

或者查看 `创建临时图标.md` 获取更多方法。

### 步骤 3: 构建项目 (1 分钟)

```bash
npm run build
```

这会在 `dist/` 目录生成可加载的扩展程序。

### 步骤 4: 加载到 Chrome (1 分钟)

1. 打开 Chrome，访问 `chrome://extensions/`
2. 开启右上角的"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择项目的 `dist` 目录
5. 看到 Digest AI 图标出现在工具栏 ✅

### 步骤 5: 配置 API Key (1 分钟)

1. 点击工具栏的 Digest AI 图标
2. 点击右上角齿轮图标（⚙️）进入设置
3. 选择 AI 模型（推荐先选 Gemini）
4. 粘贴您的 API Key
5. 点击"保存设置"
6. 点击"测试连接"验证（可选）

### 步骤 6: 保存第一篇文章 🎉

1. 打开任何文章网页（比如技术博客）
2. 点击 Digest AI 图标
3. 点击"保存文章"
4. 等待 5-10 秒
5. 看到"保存成功"后，点击"在阅读库中查看"
6. 享受 AI 生成的摘要和核心观点！

## 🐛 常见问题

### Q: 扩展无法加载？

**A**: 
- 确认已运行 `npm run build`
- 选择 `dist` 目录而不是项目根目录
- 确认 `dist/manifest.json` 存在

### Q: 保存文章失败？

**A**:
- 检查是否已配置 API Key
- 确认网络连接正常
- 查看浏览器控制台的错误信息
- 确保在文章页面而不是列表页

### Q: AI 生成的摘要不准确？

**A**:
- 这取决于 AI 模型的能力
- 可以尝试不同的模型
- 未来版本会支持自定义提示词

### Q: 数据保存在哪里？

**A**:
- 所有数据存储在本地（chrome.storage.local）
- 不会上传到任何服务器
- API Key 也是本地存储，非常安全
